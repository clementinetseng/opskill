# 首存未到帳處理流程

<AgentInstruction>
當偵測到會員說「我的錢沒進來」且關鍵字包含「首存」或「第一次」時，觸發此流程。
優先級：HIGH（首存會員流失率高，需立即處理）
</AgentInstruction>

## 2. 後台核對 (Backend Verification)

登入 Admin Panel，執行以下檢查：

### Step 2.1: 檢查訂單狀態

1. 前往 **Transactions → Deposit Orders**
2. 篩選條件：
   - Username: `[會員帳號]`
   - Time Range: `[存款時間前後 30 分鐘]`
   - Amount: `[存款金額]`

**可能的狀態：**

| 狀態 | 說明 | 處理方式 |
|------|------|---------|
| `Pending` | 訂單已建立，等待銀行確認 | → 跳至 Step 2.2 |
| `Processing` | 銀行已確認，系統正在入帳 | → 告知會員「5 分鐘內到帳」 |
| `Success` | 已入帳 | → 請會員重新整理餘額 |
| `Failed` | 訂單失敗 | → 跳至 Step 3 (退款流程) |
| `Not Found` | 查無訂單 | → 跳至 Step 2.3 (銀行流水核對) |

### Step 2.2: 檢查銀行流水 (Bank Log)

如果訂單狀態為 `Pending`，需手動核對銀行流水：

1. 前往 **Finance → Bank Statements**
2. 選擇對應的收款帳戶
3. 搜尋金額與時間

**如果找到匹配記錄：**
- 點擊「Manual Match」將流水與訂單配對
- 系統會自動入帳

**如果找不到記錄：**
- 可能是會員填錯收款帳號
- 跳至 Step 3

### Step 2.3: 請會員提供轉帳截圖

## 常見問題 FAQ

**Q: 會員堅持說已經轉帳，但銀行流水查無記錄？**
A: 請會員提供完整截圖，核對收款帳號是否正確。常見錯誤：轉到舊帳號、轉到其他平台。

**Q: 首存優惠是否需要手動發放？**
A: 系統會自動發放。如果補單，需在備註欄註明「需補發首存優惠」。

**Q: 處理時效要求？**
A: 首存會員必須在 **15 分鐘內** 給予明確回覆，否則會員流失率極高。
